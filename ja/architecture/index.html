動作環境

LucaSuiteの各アプリケーションは次の環境で動作する。

  • Ruby 3.0

Linuxのコンソール環境をプライマリターゲットとしているが、Rubyが動くUnix-likeな他のOSでも動作する。

データ管理

LucaSuiteが管理する会計書類は、gitリポジトリ内のディレクトリ構成とフラットファイル群のセットを想定している。

gitの機能により、次のようなデータ管理を実現している。

  • gitコミットによる変更履歴の記録
  • コンフリクト検出による同一ファイルの変更衝突の調停
  • 鍵認証によるセキュアなリモート転送
  • リポジトリ読み書き権限による更新プロセスの限定
  • CIツールを利用した自動処理のカスタム実装

LucaSuiteの各種ツールは、機器のローカルディレクトリ内のファイル編集機能に特化している。

用途の制約

マルチユーザーの利用を全般的に想定していない。

RDBMSを使用しないため、自動バッチ処理には適している一方、複数ユーザーが同時に編集する用途をサポートしていない。
参照のみの使い方であれば、一般的なgitリポジトリのクローン操作で矛盾なくサポートできる。

また、認証はgitリポジトリ単位のアクセス制御であるため、リポジトリの利用権限がある場合にはその範囲でデータにフルアクセスできる。
ユーザーの権限に応じて書類の一部だけアクセスするような認可制御も実装しない。

ファイルフォーマット

取引記録や設定などは、おもにCSVまたはYAMLのフラットファイルとしてコードしているため、低レベル操作を行いたい場合には、テキストエディタでも編集できる。
項目定義などのセマンティクスはアプリケーションごとに独自定義しており、日常操作ではコマンドラインツールを用いて操作する前提。

git操作シナリオの例

LucaSuiteはgitの理解と習熟を前提としている。

データディレクトリのリモート操作全般についてはLucaSuiteはツールを提供せず、gitを直接実行する。
セットアップ完了しているディレクトリ内の日常操作は、典型的には次のようなコマンドによる。

# 最新の更新内容を取得する
$ git pull origin master

# データ操作履歴をリスト表示する
$ git log

# 編集操作後にメインのデータ保管サーバーに反映する
$ git push origin master

gitリポジトリ操作にはLucaSuiteに特化した点はないため、一般的なGUIソフトを用いても同じ操作が可能だろう。
データ保管サーバーなどのセットアップは、gitそのもののセットアップである。

データバックアップ・リストア

gitは分散リポジトリであり、ネットワーク経由のバックアップ・リストアを標準提供している。
